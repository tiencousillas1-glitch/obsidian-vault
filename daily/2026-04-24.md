# 2026-04-24 — Sesión Técnica

## Problema Principal
Clowy se moría cada 20-40 minutos. 29+ reinicios en pocas horas.

## Diagnóstico Completo
1. **Síntoma**: `journalctl` mostraba SIGTERM repetidos + shutdown + systemd restart
2. **Causa raíz**: OpenClaw tiene hot-reload que detecta cambios en `openclaw.json`. La memory compaction (threshold 32k tokens) reescribía el archivo cada ~20 min con sesión larga (440 msgs) → hot-reload disparaba graceful shutdown → systemd reiniciaba
3. **Causa secundaria**: El sistema `last-known-good` de OpenClaw restauraba la config vieja en cada restart si el config estaba mal

## Solución Aplicada
1. **Compaction threshold**: 32k → 200k tokens (`softThresholdTokens`)
2. **Compaction**: Mantenida ON (evita saturación de contexto — Clowy se colgaba 8 min si está OFF)
3. **BOTH archivos**: Actualizar siempre `openclaw.json` Y `openclaw.json.last-good`
4. **Watchdog crontab**: `* * * * * /home/tien/.openclaw/watchdog.sh` — revive Clowy si muere

## Clowy ↔ Hermes en Discord (Bot-to-Bot)
- **Problema**: Hermes enviaba mensajes pero Clowy no respondía (mensajes descartados silenciosamente)
- **Causa**: `allowBots: false` por defecto en Discord de OpenClaw
- **Fix**: `"allowBots": "mentions"` en `channels.discord` (top-level, NO dentro de guild/channel)
- **Mention**: usar ID numérico `<@1491261842025218229>` — `@Clowy` no resuelve en mensajes bot→bot
