## Wolf Independent Gateway — SOLVED ✅

**Problem:** OpenClaw's anti-duplicate gateway protection prevented running Wolf as a separate instance from Clowy on the same host.

**Solution discovered:** Set `stateDir` in Wolf's config to a custom path (`/home/tien/.openclaw-wolf/state`). This creates lock files under a different `/tmp` subdirectory, bypassing the anti-duplicate detection.

**Architecture achieved:**
- Clowy: PID 1749908, port 18789, bot `8669150032` (@Clowy_97_bot)
- Wolf: PID 1752328, port 18795, bot `8753093152` (@wolf_salesbot)
- Both fully independent — separate workspaces, configs, and processes

**Key technical finding:** OpenClaw stores gateway lock files in `/tmp/openclaw-*/gateway.*.lock`. The lock key is based on system hostname, so two instances on the same machine conflict. The `stateDir` workaround creates a new lock namespace.

**Telegram 409 conflicts:** When multiple processes poll the same bot token, Telegram returns 409. OpenClaw auto-retries with exponential backoff. Conflicts clear automatically after ~1-2 minutes once all old polling sessions exit.
